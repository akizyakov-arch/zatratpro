# ТЗ: Просмотр и выгрузка документов и сканов для manager

## Цель

Реализовать для manager удобный просмотр всех документов компании и пакетную выгрузку сканов документов.

Задачи:
- manager должен уметь быстро просматривать документы компании в боте;
- manager должен уметь выгружать сканы документов пакетом;
- решение не должно ломать текущий storage;
- выгрузка должна работать по фильтрам, а не только в режиме "всё подряд".

## Текущая база

- source-файлы уже лежат в `storage/documents/<company_id>/<document_id>/source.jpg`;
- метаданные документов уже есть в БД;
- просмотр одного документа уже реализован;
- Excel-выгрузка отчетов уже реализована.

## Этап 1. Единый просмотр документов компании

Добавить новый раздел:
- `Отчеты -> Все документы`

### Поведение

1. Manager открывает `Все документы`.
2. Выбирает период.
3. Получает список документов компании.

### Поддерживаемые периоды

Быстрые периоды:
- неделя
- месяц
- квартал
- полгода
- год
- всё время

Абсолютные периоды:
- выбрать год
- выбрать месяц
- произвольный диапазон дат `с` / `по`

Требование:
- manager в 2027 или 2028 году должен иметь возможность выгрузить документы за 2026 год или любой другой прошлый период.

### Формат кнопки документа

- `дата внесения | сумма | поставщик | сотрудник`

### Ограничение списка

- первые `20` документов;
- сортировка по `created_at desc`.

### Экран карточки документа

Показывать:
- проект;
- контрагент;
- дата документа;
- номер;
- сумма;
- внес;
- дата ввода;
- первая позиция.

### Кнопки карточки

- `Открыть документ`
- `Состав документа`
- `Назад к документам`
- `Назад к отчетам`

## Этап 2. Фильтры внутри просмотра

После базового списка добавить фильтры:
- по проекту;
- по сотруднику;
- по периоду.

### UI

1. `Отчеты -> Все документы`
2. `Период`
3. `Фильтр`
- все проекты
- конкретный проект
- все сотрудники
- конкретный сотрудник

### Минимальный безопасный срез

Сначала реализовать:
- `период + проект`

Потом добавить:
- `сотрудника`

## Этап 3. Просмотр source-файла

Для всех документов в `Все документы` использовать уже существующий source-view:
- отправка как `document`;
- не как `photo`.

### Требование

- открывается только один конкретный файл;
- без fallback на legacy paths;
- только через `storage_key` и актуальный storage.

## Этап 4. Выгрузка сканов

Добавить новый раздел:
- `Отчеты -> Выгрузка сканов`

### Поток

1. Manager выбирает период.
2. При необходимости выбирает фильтр:
- все документы
- по проекту
- по сотруднику
3. Бот создает отдельную временную директорию export-job.
4. Бот собирает архив `.zip`.
5. Бот отправляет архив в чат.
6. Бот удаляет временные файлы после отправки.

## Этап 5. Формат архива

### Структура

Внутри архива:
- папка `files/`
- файл `manifest.xlsx`

### Имена файлов

Для текущего этапа:
- `files/<document_id>.jpg`

### Содержимое `manifest.xlsx`

Поля:
- `document_id`
- `project_name`
- `uploaded_by_name`
- `created_at`
- `document_date`
- `document_number`
- `vendor`
- `total_amount`
- `duplicate_status`
- `file_name_in_archive`

## Этап 6. Ограничения и защита

1. Не разрешать выгрузку без фильтра за `всё время`, если документов слишком много.
2. Ввести лимиты:
- не больше `500` документов за одну выгрузку;
- или не больше `100 MB` архива.
3. Если лимит превышен, показывать сообщение:
- `Слишком большой объем выгрузки. Уточните период или фильтр.`
4. Каждый export-job должен использовать отдельную уникальную временную директорию.
5. Параллельные выгрузки разных manager не должны пересекаться.
6. Повторный запуск одной и той же выгрузки не должен перезаписывать чужой архив.

## Этап 7. Backend

Нужно реализовать:
1. выборку документов компании по фильтрам;
2. выборку `storage_key` для source-файлов;
3. создание уникальной временной директории export-job, например по `uuid4()`;
4. сбор zip во временной директории;
5. генерацию `manifest.xlsx`;
6. отправку архива через Telegram как `document`;
7. cleanup временного zip, manifest и рабочей директории в блоке `finally` после отправки или ошибки.

## Этап 8. SQL и сервисы

Понадобятся:
1. новый view-service метод:
- `list_company_documents(...)`
2. новый view-service метод:
- `list_company_document_sources(...)`
3. новый export-service:
- `build_document_scans_archive(...)`

### Размещение

Не смешивать с существующим builder отчетов.

Создать отдельный сервис:
- `app/services/document_exports.py`

## План по фазам

### Phase 1
Базовый просмотр документов компании

1. Добавить в `Отчеты` новый пункт `Все документы`.
2. Добавить выбор быстрых периодов: неделя, месяц, квартал, полгода, год, всё время.
3. Сделать список документов компании за период с лимитом `20` и сортировкой `created_at desc`.
4. Формат кнопки документа: `дата внесения | сумма | поставщик | сотрудник`.
5. Сделать карточку документа.
6. Добавить кнопки `Открыть документ`, `Состав документа`, `Назад к документам`, `Назад к отчетам`.
7. Открытие source только как `document`.

### Phase 2
Базовая выгрузка сканов

1. Добавить в `Отчеты` новый пункт `Выгрузка сканов`.
2. Использовать быстрые периоды.
3. Собирать `zip` и `manifest.xlsx`.
4. Использовать отдельную temp-директорию на каждый export-job.
5. Удалять временные файлы в `finally`.
6. Добавить лимиты на число документов и размер архива.

### Phase 3
Произвольный период

1. Добавить кнопку `Произвольный период`.
2. Реализовать ввод двух дат сообщениями: `YYYY-MM-DD` и `YYYY-MM-DD`.
3. Сделать pending-state для ввода дат.
4. Подключить диапазон к `Все документы` и `Выгрузка сканов`.

### Phase 4
Фильтры

1. Добавить фильтр по проекту.
2. Добавить фильтр по сотруднику.
3. Поддержать комбинации период + проект + сотрудник.

### Phase 5
Усиление UX и надежности

1. Добавить progress/status message на время сборки архива.
2. Добавить логирование export-job.
3. Добавить обработку ошибок сборки архива.
4. Добавить smoke-тесты на просмотр и экспорт.

## Этап 10. Порядок реализации

1. Добавить `Отчеты -> Все документы`.
2. Сделать список документов по периоду.
3. Добавить абсолютный выбор периода: год, месяц, диапазон дат.
4. Сделать карточку документа, состав и открытие документа.
5. Добавить фильтр по проекту.

7. Добавить `Выгрузка сканов`.
8. Добавить `manifest.xlsx`.
9. Добавить уникальные export-job директории.
10. Добавить cleanup временных файлов в `finally`.
11. Добавить лимиты и сообщения об ограничениях.

## Acceptance criteria

1. Manager открывает `Отчеты -> Все документы`.
2. Выбирает период.
3. Видит список документов компании.
4. Открывает карточку документа.
5. Открывает source-файл.
6. Открывает состав документа.
7. Manager запускает `Выгрузка сканов`.
8. Получает zip-архив.
9. Внутри архива есть сканы и `manifest.xlsx`.
10. Документы вне компании в выборку не попадают.
