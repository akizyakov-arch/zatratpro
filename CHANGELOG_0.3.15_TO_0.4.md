# CHANGELOG 0.3.15 -> 0.4

## Пользовательские изменения
- Invite-коды бот теперь чаще отправляет отдельным сообщением, а не внутри длинного текста. Копировать удобнее.
- После принятия invite бот сразу досылает новое главное меню с уже актуальной ролью и доступами.
- При назначении в компанию пользователь получает уведомление.
- При исключении из компании пользователь получает уведомление.
- При исключении очищаются `pending_actions` и `pending_documents`, чтобы пользователь не продолжал старый flow.
- После исключения бот показывает уже новое меню без company-access.

## Owner / Manager
- Вернули owner-разделение компаний на активные и архивированные.
- Починили owner-назначение пользователя в компанию.
- Починили owner-исключение пользователя из компании.
- Починили manager-исключение так, чтобы после исключения реально снимался доступ.
- Починили создание manager/employee invite под новую схему invite-статусов и `start_token`.

## Документы / OCR
- Перед OCR фото теперь мягко нормализуется:
  - поворот по EXIF
  - JPEG
  - ограничение ширины
  - умеренное сжатие
- Добавлен retry OCR при timeout и `E101`.
- В duplicate-confirm flow состояние теперь сохраняется корректно, поэтому кнопка `Все равно добавить` не теряет документ.
- `pending_documents` и `pending_actions` больше не в памяти процесса, а в Postgres, поэтому flow переживает рестарт бота.

## Меню / Скорость
- `start.py` сильно облегчён: из него вынесены owner, manager, reports, help, onboarding и pending text flow.
- Убраны лишние обращения в БД в hot paths меню.
- `Помощь` внутри callback-ов больше не ходит в БД.
- `callback.answer()` в help/report/menu ветках теперь отправляется раньше, поэтому кнопки ощущаются быстрее.
- Исправлены callback-path’ы, где context ошибочно читался от `callback.message`, а не от пользователя.

## Статусы и доступ
- Invite-flow переведен на схему статусов `new / used / expired / revoked`.
- Исправлены места, где путались статусы `company_invites` и `company_members`.
- Починены join/access сценарии для manager и employee после invite.

## Архитектурные изменения
- Декомпозирован handler-слой:
  - `owner.py`
  - `manager.py`
  - `reports.py`
  - `help.py`
  - `onboarding.py`
  - `common_menu.py`
- Добавлен `AccessContext` / `AccessService` для более чистого доступа к menu state.
